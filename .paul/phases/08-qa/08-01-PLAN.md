## T2 — Triage Windows failures
type: auto

### Files
- MODIFY TESTING.md (append triage table to a new ## Triage section)

### Action

Read all `[F]` entries in the Windows columns of TESTING.md.
For each failure found:
- Assign severity: P0 / P1 / P2 (criteria below)
- Note the test ID, description, and severity

**Severity criteria:**
- **P0** — crash, hang, data loss, audio blowup, or plugin fails to load
- **P1** — feature completely non-functional with no workaround (e.g., preset load fails, a whole module produces no effect)
- **P2** — cosmetic issue, partial degradation with a workaround, or test marginal (e.g., CPU slightly over budget, minor UI label wrong)

Append a `## Triage` section at the end of TESTING.md (before the sign-off block):

```markdown
## Triage — Windows pass (v1.0.0-beta.1)

| ID | Description | Platform | Severity | Status |
|----|-------------|----------|----------|--------|
| T{n}.{m} | description | Win VST3 / Win SA | P0/P1/P2 | Open |
```

If zero failures: append `## Triage — Windows pass: ✅ CLEAN — zero failures`

Commit:
```bash
git add TESTING.md
git commit -m "test(qa): triage Windows failures — v1.0.0-beta.1"
```
