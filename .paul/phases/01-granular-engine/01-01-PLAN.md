## Goal

Implement the three RT-safe data structures that form the foundation of the granular engine: `EnvelopeShapes` (5 window functions), `Grain` (single grain descriptor, zero-allocation), and `GrainPool` (pre-allocated 1024-slot pool with lock-free acquire/release). Add Catch2 unit tests covering all three.

## Purpose

The engine in plan 01-02 depends on these types being correct and verifiably RT-safe before threading is introduced. Testable in complete isolation — no audio thread, no FIFO, no workers. Validates DSP math and pool invariants before complexity increases.

  <name>Write EnvelopeShapes.h + EnvelopeShapes.cpp</name>
  <files>Source/Engine/EnvelopeShapes.h, Source/Engine/EnvelopeShapes.cpp</files>
  <action>
    Create `enum class EnvelopeShape { Hann, Tukey, Gaussian, Triangle, Rectangle };` in the header.

    Provide a single dispatch function:
    ```cpp
    float applyEnvelope(EnvelopeShape shape, float phase) noexcept;
    ```
    `phase` is in [0, 1]. Return value is in [0, 1]. Function is `noexcept` — no exceptions on audio thread.

    Implement in EnvelopeShapes.cpp:
    - **Hann**: `0.5f * (1.0f - std::cos(juce::MathConstants<float>::twoPi * phase))`
    - **Tukey (α=0.5)**: flat centre with cosine taper at each end (α/2 of total length). If phase < α/2: cosine taper up. If phase > 1 - α/2: cosine taper down. Otherwise 1.0f.
    - **Gaussian (σ=0.4 of half-window)**: `std::exp(-0.5f * std::pow((phase - 0.5f) / (0.4f * 0.5f), 2.0f))`. Clamp output to [0, 1].
    - **Triangle**: linear ramp up to 0.5, linear ramp down from 0.5. `phase < 0.5f ? 2.0f * phase : 2.0f * (1.0f - phase)`.
    - **Rectangle**: return 1.0f for all phase values.

    Avoid: `std::cos(phase * M_PI)` — use `juce::MathConstants<float>::pi` instead for portability.
    Avoid: branching on float equality — use the mathematical formula directly.

    Include `<juce_core/juce_core.h>` for `juce::MathConstants`.
    Header include guard: `#pragma once`.
  </action>
  <verify>Unit tests in AC-1 pass. Manual spot-check: `applyEnvelope(EnvelopeShape::Hann, 0.0f)` ≈ 0, `applyEnvelope(EnvelopeShape::Hann, 0.5f)` ≈ 1.</verify>
  <done>AC-1 satisfied: all 5 shapes produce correct values at boundary and midpoint.</done>
</task>

<task type="auto">
  <name>Write Grain.h + GrainPool.h + GrainPool.cpp</name>
  <files>Source/Engine/Grain.h, Source/Engine/GrainPool.h, Source/Engine/GrainPool.cpp</files>
  <action>
    **Grain.h** — plain data struct, trivially copyable:
    ```cpp
    #pragma once
    #include "EnvelopeShapes.h"

    struct Grain
    {
        const float* source      = nullptr;  // non-owning; points into SampleBuffer
        int          startPos    = 0;        // sample index in source
        int          lengthSamples = 0;
        float        pitchRatio  = 1.0f;     // 1.0 = no pitch shift
        float        pan         = 0.0f;     // -1..+1
        EnvelopeShape shape      = EnvelopeShape::Hann;
        float        currentPhase = 0.0f;   // 0..1 playback position
    };
    static_assert(std::is_trivially_copyable_v<Grain>, "Grain must be trivially copyable");
    ```
    No constructor, no destructor — aggregate init only. No `std::atomic` members on Grain itself.

    **GrainPool.h**:
    ```cpp
    #pragma once
    #include "Grain.h"
    #include <array>
    #include <atomic>

    class GrainPool
    {
    public:
        static constexpr int Capacity = 1024;

        GrainPool() noexcept;

        // RT-safe: bounded scan, no allocation, no locks.
        // Returns nullptr if pool exhausted.
        Grain* acquire() noexcept;

        // RT-safe: clears slot atomically.
        void release(Grain* grain) noexcept;

        int activeCount() const noexcept;

    private:
        std::array<Grain, Capacity>            slots_;
        std::array<std::atomic<bool>, Capacity> inUse_;
        std::atomic<int>                        hint_{0};  // scan start hint
    };
    ```

    **GrainPool.cpp** — implement acquire/release:
    - `acquire()`: start scan from `hint_`. Loop through all `Capacity` slots. For each slot, try `inUse_[i].compare_exchange_strong(false, true, std::memory_order_acquire, std::memory_order_relaxed)`. On success: update `hint_` to `(i + 1) % Capacity`, zero-initialise `slots_[i]`, return `&slots_[i]`. If full loop completes without success: return nullptr.
    - `release(Grain* grain)`: compute index as `grain - slots_.data()`. Assert index in [0, Capacity). Set `inUse_[index].store(false, std::memory_order_release)`.
    - `activeCount()`: count slots where `inUse_[i].load(std::memory_order_relaxed)` is true.

    Avoid: `std::mutex`, `new`, `delete`, `std::vector`, any STL that allocates.
    Avoid: `assert()` in release builds on the audio thread — use `jassert()` which compiles out in release.
  </action>
  <verify>Pool overflow test in test_engine.cpp: acquire 1024, verify 1025th returns nullptr, release one, verify next acquire succeeds.</verify>
  <done>AC-2 and AC-3 satisfied: pool manages exactly 1024 slots, Grain is trivially copyable.</done>
</task>
