<objective>
**Goal**: Establish a buildable, validated JUCE 8 plugin shell for Grano. No DSP, no UI, no parameters — just the scaffold that produces working VST3, AU, and Standalone artifacts on Windows, macOS, and (best-effort) Linux, with CI green.

**Purpose**: De-risk the build system, the JUCE integration, the plugin format generation, and the CI matrix before writing any audio code. Every subsequent phase depends on this scaffold.

**Output**: A repo where `cmake -B build && cmake --build build` produces working plugins on every supported platform, validated by `pluginval` at strictness 5, with a passing CI workflow on `main`.
</objective>

<acceptance_criteria>

## AC-1: CMake produces all three plugin formats on every primary platform

**Given** a clean clone of the repo and JUCE 8 at `~/JUCE`
**When** the developer runs `cmake -B build -DCMAKE_BUILD_TYPE=Debug && cmake --build build --config Debug`
**Then** the `build/` directory contains:
- `Grano_artefacts/Debug/VST3/Grano.vst3/` (all platforms)
- `Grano_artefacts/Debug/AU/Grano.component/` (macOS only)
- `Grano_artefacts/Debug/Standalone/Grano(.exe|.app|<bin>)` (all platforms)

## AC-2: Standalone runs and shows a window

**Given** a Debug build completed
**When** the developer launches the Standalone artifact
**Then** a window appears titled "Grano" with no controls beyond the JUCE default menu. The application does not crash within 30 seconds of idle.

## AC-3: pluginval passes at strictness 5

**Given** a Debug VST3 build on Windows or macOS
**When** `pluginval --strictness-level 5 --validate-in-process path/to/Grano.vst3` runs
**Then** exit code is 0 with no errors. Warnings are acceptable.

## AC-4: CI workflow passes on Windows and macOS

**Given** a push to the `main` branch
**When** GitHub Actions runs the workflow defined in `.github/workflows/ci.yml`
**Then** both the `windows-latest` and `macos-latest` jobs complete successfully, each producing the three plugin artifacts as workflow artifacts.

## AC-5: At least one Catch2 test passes

**Given** the test runner is built
**When** `ctest --test-dir build --output-on-failure` runs
**Then** at least one test case named `"Plugin name is Grano"` passes.

## AC-6: Linux secondary target builds without errors

**Given** Ubuntu 22.04+ with required dependencies installed (per `BUILD.md`)
**When** the same CMake commands run as on the primary platforms
**Then** the VST3 and Standalone artifacts are produced. (AU is macOS-only, expected absent.) Linux is best-effort: failures may be filed as deferred issues but do not block AC-1 through AC-5.

## AC-7: Project structure matches PROJECT.md conventions

**Given** a fresh clone after this plan completes
**When** inspecting the file tree
**Then** the following directories exist (some empty, ready for population):
- `Source/`
- `Source/Engine/`
- `Source/Modules/`
- `Source/Modulation/`
- `Source/UI/`

- `Tests/`
And these files exist:

- `Tests/test_main.cpp`
- `.github/workflows/ci.yml`

<task type="auto">
  <name>Write top-level CMakeLists.txt</name>
  <files>CMakeLists.txt</files>
  <action>
    Create `CMakeLists.txt` at the repository root with these properties:

    - `cmake_minimum_required(VERSION 3.22)`
    - `project(Grano VERSION 0.1.0 LANGUAGES CXX C)`
    - C++ standard 20, required, no extensions.
    - `set(JUCE_DIR "$ENV{HOME}/JUCE" CACHE PATH "Path to JUCE clone")` with a fallback for `%USERPROFILE%` on Windows.
    - `add_subdirectory(${JUCE_DIR} JUCE-build EXCLUDE_FROM_ALL)`.
    - `juce_add_plugin(Grano ...)` with:
      - `PRODUCT_NAME "Grano"`
      - `COMPANY_NAME "Bidu"` (placeholder; can be edited)
      - `BUNDLE_ID "br.com.bidu.grano"`
      - `PLUGIN_MANUFACTURER_CODE "Bidu"`
      - `PLUGIN_CODE "Gran"`
      - `FORMATS VST3 AU Standalone` (AU only on Apple platforms — use `if(APPLE) ... endif()` to gate)
      - `IS_SYNTH FALSE` (it's an effect-like instrument that loads samples; revisit if MIDI triggering changes scope)
      - `NEEDS_MIDI_INPUT TRUE` (for MIDI learn later, harmless now)
      - `EDITOR_WANTS_KEYBOARD_FOCUS TRUE`
      - `VST3_CATEGORIES "Instrument" "Sampler"`
      - `AU_MAIN_TYPE "kAudioUnitType_MusicDevice"`
    - `target_sources(Grano PRIVATE Source/PluginProcessor.cpp Source/PluginEditor.cpp)`.
    - `target_compile_definitions(Grano PRIVATE JUCE_WEB_BROWSER=0 JUCE_USE_CURL=0 JUCE_VST3_CAN_REPLACE_VST2=0 JUCE_USE_MP3AUDIOFORMAT=1)`.
    - `target_link_libraries(Grano PRIVATE juce::juce_audio_utils juce::juce_audio_processors juce::juce_dsp juce::juce_gui_extra)`.
    - `target_link_libraries(Grano PRIVATE juce::juce_recommended_config_flags juce::juce_recommended_lto_flags juce::juce_recommended_warning_flags)`.
    - Enable testing: `enable_testing()`, `add_subdirectory(Tests)`.

    Inline-comment every section with a one-line explanation of why each setting is there.
  </action>
  <verify>`cmake -B build -DCMAKE_BUILD_TYPE=Debug` succeeds without errors. Inspect `build/` for the generated build files.</verify>

<boundaries>

## DO NOT CHANGE in this plan

- Do not add any DSP code. The processor's `processBlock` clears the buffer and nothing else.
- Do not add any parameters or APVTS. F3 is the parameter phase.
- Do not add UI controls beyond the F0 label. The editor in F0 is a placeholder.
- Do not modify `.paul/PROJECT.md`, `.paul/ROADMAP.md`, or `.paul/SPECIAL-FLOWS.md`. Decisions there are settled.
- Do not introduce dependencies beyond JUCE 8 and Catch2 v3. Anything else needs an ADR added in `PROJECT.md` first.

## Out of scope in this plan

- Code signing and notarization (deferred to F7).
- Release builds and installers (F7).
- MIDI handling (F5).
- Sample loading (F2).

</boundaries>
