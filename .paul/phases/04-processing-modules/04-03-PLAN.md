# PLAN — 04-03: Multi-platform CI & build hardening

**Phase:** F4 — Processing Modules (infrastructure sub-plan)
**Created:** 2026-05-18
**Status:** awaiting approval

## Context

CI and CMake config have 5 gaps that risk unpredictable failures as the project distributes on Windows and macOS:

1. **JUCE cloned from `--branch master`** — any breaking upstream commit silently breaks CI; version is undocumented
2. **macOS missing `CMAKE_OSX_DEPLOYMENT_TARGET`** — JUCE defaults to older SDK; Apple Silicon + Xcode 15 require explicit 11.0+
3. **JUCE cache key lacks version string** — stale cache won't bust when JUCE is upgraded
4. **Linux jobs skip JUCE caching** — `linux-best-effort` and `tsan` both re-clone JUCE every run (~90s wasted per job)
5. **Linux builds are serial** — no `-j$(nproc)` on Linux cmake --build steps

Additionally, `CMakeLists.txt` lacks the macOS deployment target and Windows `/utf-8` flag for local dev reproducibility.

**Scope:** `.github/workflows/ci.yml` + `CMakeLists.txt` only. No DSP or plugin source changes.

## Acceptance criteria

- [ ] JUCE version defined once (workflow `env:` var); all 3 jobs clone same pinned tag
- [ ] No `--branch master` remaining for JUCE clones in ci.yml
- [ ] macOS configure step (CI) passes `-DCMAKE_OSX_DEPLOYMENT_TARGET=11.0`
- [ ] CMakeLists.txt sets `CMAKE_OSX_DEPLOYMENT_TARGET=11.0` default for local macOS builds
- [ ] CMakeLists.txt adds `/utf-8` for MSVC (Windows source encoding)
- [ ] Both Linux jobs have JUCE cache + conditional clone (same pattern as `build` job)
- [ ] Linux build steps pass `-j$(nproc)`
- [ ] CI green on push to `main`

## Tasks

### Task 1 — Pin JUCE version in CMakeLists.txt (macOS target + MSVC /utf-8)

**Files:** `CMakeLists.txt`

**Action:**

After the `CMAKE_CXX_EXTENSIONS OFF` block, insert:

```cmake
# ── Platform compile flags ────────────────────────────────────────────────────
if(APPLE)
    # Big Sur minimum: first release with Apple Silicon + satisfies Xcode 15 requirements.
    set(CMAKE_OSX_DEPLOYMENT_TARGET "11.0" CACHE STRING "Minimum macOS deployment target")
endif()

if(MSVC)
    add_compile_options(/utf-8)  # enforce UTF-8 source/exec charset on Windows
endif()
```

**Verify:**
- `grep -n "OSX_DEPLOYMENT_TARGET" CMakeLists.txt` → line present
- `grep -n "utf-8" CMakeLists.txt` → line present

**Done:** Both flags in file.

---

### Task 2 — Pin JUCE tag in CI (workflow env var + cache key)

**Files:** `.github/workflows/ci.yml`

**Action:**

Add workflow-level `env:` block at top (after `on:`):
```yaml
env:
  JUCE_VERSION: "8.0.7"
```

Replace all 3 `--branch master` JUCE clone lines with:
```
git clone --depth 1 --branch ${{ env.JUCE_VERSION }} https://github.com/juce-framework/JUCE.git ~/JUCE
```

Update `build` job JUCE cache key from:
```
key: juce-8-${{ runner.os }}-${{ hashFiles('**/CMakeLists.txt') }}
```
to:
```
key: juce-${{ env.JUCE_VERSION }}-${{ runner.os }}-${{ hashFiles('**/CMakeLists.txt') }}
```

**Verify:** `grep -n "branch master" .github/workflows/ci.yml` → 0 hits

**Done:** All clone steps use `${{ env.JUCE_VERSION }}`. Cache key includes version.

---

### Task 3 — Add macOS deployment target to CI configure step

**Files:** `.github/workflows/ci.yml`

**Action:**

In the "Configure (macOS)" step, add `-DCMAKE_OSX_DEPLOYMENT_TARGET=11.0`:

```yaml
- name: Configure (macOS)
  if: runner.os == 'macOS'
  run: |
    cmake -B build -G Xcode \
      -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_OSX_DEPLOYMENT_TARGET=11.0 \
      -DJUCE_DIR="$HOME/JUCE"
```

**Verify:** `grep -n "OSX_DEPLOYMENT" .github/workflows/ci.yml` → match in macOS configure step.

**Done:** Flag present in ci.yml macOS configure.

---

### Task 4 — Cache JUCE on Linux jobs

**Files:** `.github/workflows/ci.yml`

**Action:**

In both `linux-best-effort` and `tsan` jobs, add JUCE cache steps immediately before "Clone JUCE 8" and make the clone conditional, mirroring the `build` job pattern:

```yaml
- name: Cache JUCE
  id: cache-juce
  uses: actions/cache@v4
  with:
    path: ~/JUCE
    key: juce-${{ env.JUCE_VERSION }}-${{ runner.os }}-${{ hashFiles('**/CMakeLists.txt') }}

- name: Clone JUCE 8
  if: steps.cache-juce.outputs.cache-hit != 'true'
  run: git clone --depth 1 --branch ${{ env.JUCE_VERSION }} https://github.com/juce-framework/JUCE.git ~/JUCE
```

**Verify:** Both Linux jobs show the cache + conditional clone pattern.

**Done:** Visual inspection confirms both jobs updated.

---

### Task 5 — Parallel builds on Linux

**Files:** `.github/workflows/ci.yml`

**Action:**

In `linux-best-effort` Build step, add parallel flag:
```yaml
- name: Build
  run: cmake --build build --config Release -- -j$(nproc)
```

In `tsan` Build step (already has `-j$(nproc)` on target line — confirm it's correct):
```yaml
- name: Build GranoTests
  run: cmake --build build --target GranoTests -j$(nproc)
```

**Verify:** `grep -n "nproc" .github/workflows/ci.yml` → ≥2 hits (linux-best-effort Build + tsan Build).

**Done:** Both Linux build steps use `-j$(nproc)`.
