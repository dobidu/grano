## What Was Done This Session

- **Windows drag-drop properly fixed** — root-caused and replaced. JUCE registers an OLE
  `IDropTarget`, but OLE drops only deliver on a shared STA; the prior `WM_DROPFILES` hack was a
  no-op (JUCE's wndproc ignores that message) and an interim `RevokeDragDrop` attempt caused a
  "forbidden cursor everywhere" regression. Final fix: a custom `GranoDropTarget : IDropTarget`
  (`PluginEditor.cpp`) that returns `DROPEFFECT_COPY` and extracts `CF_HDROP` via
  `DragQueryFileW`, registered with `RegisterDragDrop` on JUCE's STA message thread (retried each
  10 Hz timer tick), revoked + released in the dtor. **Verified working** on Windows Standalone
  (WAV + MP3). Commit `154d23e`.
- **README** — added `screenshot.png` near top with caption; fixed stale Status table
  (F7 ✅ done / F8 🔄 in progress). Screenshot later updated by user. Commits `d8facba`, `0757c30`.

## Important Context

- **Release mechanism:** push a `v*` tag → CI builds Win+macOS Release, runs tests + pluginval-10,
  packages `Grano-<tag>-{Windows,macOS}.zip` (VST3/AU/Standalone + LICENSE + README), creates a
  GitHub `--prerelease`. Linux is best-effort, NOT packaged. macOS AU is unsigned (Gatekeeper
  `xattr` note in README/release body).
- **To re-cut an existing tag:** delete the GitHub release (`gh release delete`), delete tag
  local+remote, re-tag at the new commit, push tag. `gh release create` fails if the release
  already exists, so the old one must be deleted first.
- **Drag-drop:** custom OLE `IDropTarget`, not `WM_DROPFILES` — see fix above. `masterVolume` is
  already in dB (−60..+6); never call `gainToDecibels()` on it. Pitch +24st bounds-checked.
- **CI deprecation warnings only** (Node 20 actions, `windows-latest`→`windows-2025` redirect) —
  non-blocking; future maintenance, not urgent.
